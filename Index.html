React.js vs Next.js

ðŸ¦– React.js

Rendering▼
Client-Side Rendering (CSR) secara default. Artinya, browser mengunduh file JavaScript minimal, lalu JavaScript tersebut mengambil alih untuk merender seluruh halaman. Pengguna mungkin melihat halaman kosong sesaat.
Routing▼
Tidak ada bawaan. Kamu perlu menginstal dan mengkonfigurasi library tambahan seperti React Router untuk mengatur navigasi antar halaman di dalam aplikasi (Single Page Application).
Data Fetching▼
Dilakukan di sisi klien (browser). Biasanya menggunakan hook useEffect dan fetch API atau library seperti Axios. Data diambil setelah komponen di-render pertama kali.
SEO▼
Kurang optimal secara default. Karena konten di-render oleh JavaScript, mesin pencari seperti Google harus menjalankan JavaScript untuk mengindeks halaman, yang bisa lebih lambat dan kurang efektif.

ðŸš€ Next.js

Rendering▼
Hybrid Rendering. Mendukung Server-Side Rendering (SSR) dan Static Site Generation (SSG) secara bawaan. Halaman HTML sudah jadi saat dikirim dari server, sehingga lebih cepat tampil dan bagus untuk SEO. Bisa juga CSR per komponen.
Routing▼
Sudah termasuk (built-in) dan berbasis file-system. Cukup buat file di dalam folder /pages atau /app, dan Next.js akan otomatis membuatkan rute untukmu. Sangat mudah dan cepat.
Data Fetching▼
Sangat fleksibel. Kamu bisa mengambil data di server sebelum halaman di-render (dengan getServerSideProps atau React Server Components) atau saat proses build (dengan getStaticProps). Ini membuat data sudah tersedia saat halaman dimuat.
SEO▼
Sangat SEO-friendly. Karena halaman sudah di-render di server (SSR/SSG), mesin pencari bisa langsung membaca konten HTML lengkap tanpa perlu menjalankan JavaScript. Ini adalah keunggulan utama Next.js.

ðŸ’¡ Kapan Pakai yang Mana?

Gunakan React.js jika kamu membangun aplikasi web yang sangat interaktif dan tidak terlalu bergantung pada SEO, seperti dashboard internal atau aplikasi yang dilindungi login.

Gunakan Next.js jika kamu butuh performa load yang cepat, SEO yang kuat, atau keduanya. Sangat ideal untuk website publik seperti blog, situs e-commerce, dan landing page.

Bedah Kode: Bagaimana Halaman Ini Dibuat?

ðŸŽ¨ CSS: Desain Visual & Styling ▼
Bagian CSS di dalam tag <style> bertanggung jawab atas semua tampilan visual halaman ini.
  • @import url(...): Baris pertama ini mengimpor font 'Poppins' dari Google Fonts agar teks terlihat modern dan enak dibaca.
  • body: Mengatur dasar halaman, seperti warna latar belakang (#f0f2f5), jenis font, dan menggunakan Flexbox (display: flex) untuk menempatkan konten utama di tengah layar.
  • .container: Memberi area konten utama sebuah latar belakang putih, sudut tumpul (border-radius), dan bayangan (box-shadow) agar terlihat "melayang".
  • .comparison-grid: Ini adalah inti dari layout perbandingan. display: grid dan grid-template-columns: 1fr 1fr membaginya menjadi dua kolom yang sama lebarnya.
  • .feature-card: Ini adalah class untuk setiap kartu yang bisa diklik. Diberi bayangan, sudut tumpul, dan cursor: pointer agar pengguna tahu ini bisa diinteraksi.
  • Animasi Accordion: Triknya ada di sini.
    • .card-content diberi max-height: 0 dan overflow: hidden. Ini membuatnya "terlipat" atau tidak terlihat.
    • Ketika JavaScript menambahkan class .active pada kartu, aturan .feature-card.active .card-content diterapkan. Aturan ini mengubah max-height menjadi nilai yang lebih besar (misal: 500px), sehingga kontennya "terbuka".
    • transition: max-height 0.5s ease-out membuat perubahan tinggi ini berjalan mulus selama 0.5 detik, bukan terjadi tiba-tiba.
âœ¨ JavaScript: Otak & Interaktivitas ▼
Kode JavaScript di dalam tag <script> sangat singkat namun powerful. Tujuannya hanya satu: mengelola buka-tutup kartu.
  • function toggleCard(cardElement): Ini adalah fungsi yang dipanggil setiap kali sebuah kartu diklik. cardElement adalah kartu spesifik yang baru saja diklik oleh pengguna.
  • Menutup Kartu Lain:
    • const column = cardElement.closest('.framework-column');: Mencari "kolom induk" dari kartu yang diklik.
    • const allCards = column.querySelectorAll('.feature-card');: Mengambil semua kartu yang ada di dalam kolom tersebut.
    • allCards.forEach(...): Loop ini memeriksa setiap kartu di dalam kolom. Jika ada kartu lain yang sedang terbuka (memiliki class .active) dan itu bukan kartu yang baru saja kita klik, maka class .active-nya dihapus, sehingga kartu itu tertutup. Ini memastikan hanya satu kartu yang bisa terbuka dalam satu waktu di setiap kolom.
  • cardElement.classList.toggle('active');: Ini adalah perintah utamanya. .toggle() bekerja seperti saklar lampu. Jika kartu belum punya class 'active', ia akan menambahkannya. Jika sudah punya, ia akan menghapusnya. CSS kemudian merespons perubahan class ini untuk menampilkan atau menyembunyikan konten.
